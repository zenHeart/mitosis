### 俄罗斯方块"不可玩"的常见原因（必须避免）

| 失败模式 | 原因 | 对应标准 |
|----------|------|---------|
| 方块不动 | 输入事件未绑定或 grid 不更新 | GM2, T2-T4 |
| 方块穿墙/穿底 | 碰撞检测缺失或错误 | GM3, T5 |
| 满行不消除 | 消行逻辑缺失或行判断错误 | GM3, T6 |
| 分数永远为 0 | 计分逻辑缺失 | GM3, T7 |
| 游戏永远不会结束 | 结束条件缺失 | GM4, T8 |
| 无法重新开始 | 重置逻辑缺失 | T9 |

## 分类型实现指南

### 游戏类应用
- **必须包含完整的游戏循环：** 初始化 → 游戏进行 → 结束判定 → 重新开始
- **必须包含输入处理：** 键盘事件（keydown）绑定到游戏操作
- **必须包含状态管理：** 分数、等级、生命值等用 reactive/ref 管理
- **必须包含视觉反馈：** 游戏区域、分数面板、状态提示
- **CSS：** 使用 CSS Grid / Flexbox 布局，深色背景 + 亮色元素，圆角边框

### 工具类应用
- **必须包含完整的功能流程：** 输入 → 处理 → 输出
- **必须包含状态持久化（可选）：** localStorage 保存数据
- **必须包含错误处理：** 空值检查、边界情况处理

### 通用质量要求
- **交互完整性：** 每个按钮/操作必须有实际效果，不能有空函数
- **视觉质量：** 使用 CSS 变量（:root）定义配色方案，hover/active 状态
- **响应式：** 使用 max-width + 居中布局，适配移动端
- **无 console.error：** 代码中不应有无意义的 console.error

## 常见陷阱（必须避免）
