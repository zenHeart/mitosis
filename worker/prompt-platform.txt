1. 读取 Issue #__ISSUE_NUMBER__ 的完整内容
2. **提取具体要求：** 从 Issue 正文中提取所有功能点、交互要求、视觉要求
3. **确认你理解了需求** — 在开始编码前，列出你理解的 3-5 个核心功能点

## 上下文
- 任务类型: 平台构建（修改 Mitosis 平台本身）
- 需求来源: Issue #__ISSUE_NUMBER__
- 工作目录: 仓库根目录（repo root）

## 铁律（违反任一导致构建失败）

1. **在仓库根目录执行所有命令** — 不要 cd 到 apps/ 子目录
2. **TypeScript strict 模式** — 所有 tsconfig.json 必须设置 "strict": true
3. **构建必须通过** — npm run typecheck 和 npm run build 都必须 0 errors, 0 warnings
4. **提交所有变更** — 修改的平台文件必须 git add 和 git commit
5. **不依赖外部 CDN** — 所有代码内联，不引入外部资源

## 平台文件修改范围

你可以修改以下平台文件：
- `src/` — 前端源码（Vue 组件、Store、Composable、类型定义）
- `docs/` — 技术文档
- `.github/` — CI 配置
- `worker/` — Cloudflare Worker
- `.claude/` — Claude Code 配置
- `package.json` — 依赖管理
- `vite.config.ts` — 构建配置

## 工作流程

1. **读取 Issue** — 完整阅读 Issue #__ISSUE_NUMBER__ 的内容
2. **列出理解** — 输出你理解的 3-5 个核心功能点
3. **实现功能** — 修改必要的平台文件
4. **运行验证** — 执行 npm run typecheck && npm run build
5. **提交变更** — git add . && git commit -m "feat: platform changes (issue #__ISSUE_NUMBER__)"
6. **确认完成** — 输出修改了哪些文件

## 禁止事项

- 禁止修改 apps/ 目录下的应用文件
- 禁止添加未在 Issue 中描述的功能
- 禁止提交调试代码或 console.log
- 禁止使用 any 类型
- 禁止破坏现有功能
