# Quality Rubric

## 等级

| 等级 | 含义 | CI 结果 |
|------|------|---------|
| P1 | 可发布：核心功能可用，无阻断错误 | 可以部署 |
| P2 | 基本可用：功能完整但体验仍可优化 | 可记录后续优化 |
| P3 | 结构正确：能构建但核心功能不足 | 不部署 |
| F | 失败：构建、加载或基础文件失败 | 不部署 |

发布门槛是 P1。

## 通用 P1

| 编号 | 标准 | 验证 |
|------|------|------|
| G1 | 应用能正常加载 | Playwright 或浏览器打开无阻断错误 |
| G2 | 核心交互有响应 | 点击、输入或按键后页面状态变化 |
| G3 | 无阻断性 runtime error | console error / pageerror 为 0 |
| G4 | 构建产物完整 | `dist/index.html` 和 JS/CSS assets 存在 |
| G5 | TypeScript strict | `vue-tsc` 0 errors |
| G6 | Vite build 成功 | `npm run build` exit 0 |

## 游戏类 P1

| 编号 | 标准 |
|------|------|
| GM1 | 游戏元素初始化后可见 |
| GM2 | 玩家输入有效 |
| GM3 | 游戏规则会改变状态 |
| GM4 | 有明确结束条件 |
| GM5 | 可重新开始 |

## 工具类 P1

| 编号 | 标准 |
|------|------|
| TM1 | 输入能产生有逻辑关系的输出 |
| TM2 | 核心操作连续执行不崩溃 |
| TM3 | 异常输入有错误状态 |

## Vite base 规则

- 平台主站：`base: '/'`。
- Agent 生成应用：`base: './'`。

## 验收原则

- Agent 自检不是验收。
- 构建通过不是可发布。
- verifier 失败必须继续修复或明确阻塞。
- 没有 fresh 验证输出，不能声明完成。
